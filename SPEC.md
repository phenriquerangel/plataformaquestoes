## 1. Business Context

### Platform Purpose
✅ DEFINED

EduQuest.ai is a web platform that enables teachers to generate, manage, and export pedagogical questions using Google Gemini AI. Teachers select a subject (matéria), topic (assunto), difficulty, quantity, and question type; the AI generates questions in streaming. Generated questions can be organized into lists and exported as PDFs or shared as public links.

### Target Users and Personas
⚠️ PARTIAL — inferred from code; no formal persona documentation exists.

| Persona | Role | Access |
|---|---|---|
| Professor | Creates and manages questions/lists | role=user |
| Admin | Full access: all questions, all lists, user management, curriculum import | role=admin |
| Aluno / Externo | View published lists via public link | No auth required |

## 2. Architecture

### Architectural Pattern
✅ DEFINED — Microservices (small scale): 4 services, each in its own container.

| Service | Technology | Port |
|---|---|---|
| backend | FastAPI + SQLAlchemy | 8000 |
| frontend | React + Vite served by Nginx | 80 |
| pdf-renderer | Node.js + Puppeteer | 3000 |
| postgres | PostgreSQL 15 Alpine | 5432 |

### Cloud Provider / Regions
❌ TO DISCUSS — manifests target a generic Kubernetes cluster. Currently runs on Docker Desktop local cluster (`desktop-control-plane`). No cloud provider defined.

### Multi-Cloud / Hybrid Strategy
❌ TO DISCUSS — not applicable at current scale.

### Kubernetes
✅ DEFINED

- **Cluster**: Single cluster, namespace `default` (workloads) + `monitoring` (observability)
- **Workloads**: 4 Deployments, all with `replicas: 1`
- **No HPA, PDB, or resource limits defined** — single-replica, no autoscaling
- **Storage**: 1 PVC for Postgres (`postgres-pvc`, 1Gi, ReadWriteOnce)
- **Ingress**: None — Nginx inside `frontend` pod proxies `/api/*` to backend service

| Deployment | Image | Namespace |
|---|---|---|
| backend-questions | backend-questions:\<tag\> | default |
| frontend-questions | frontend-questions:\<tag\> | default |
| pdf-renderer | pdf-renderer:\<tag\> | default |
| postgres | postgres:15-alpine | default |

### Terraform
❌ TO DISCUSS — no IaC. Cluster and manifests managed manually/via deploy scripts.

### Service Mesh / Networking Topology
⚠️ PARTIAL

- No service mesh (no Istio/Linkerd)
- Nginx in the frontend pod acts as the only ingress point
- Internal routing: Kubernetes DNS (`<service>.default.svc.cluster.local`)
- External access: `kubectl port-forward svc/frontend-service 8080:80` (no LoadBalancer/Ingress resource)

```
User → port-forward:8080 → frontend-service:80 → Nginx
                                                    ├── /api/* → backend-questions-service:8000
                                                    └── /*     → SPA (index.html)

backend → postgres-service:5432
backend → pdf-renderer-service:3000 (PDF export)
backend → Gemini API (external, HTTPS)
backend → tempo.monitoring.svc.cluster.local:4317 (optional tracing)
```

## 5. Security

### AuthN / AuthZ
✅ DEFINED

- **Mechanism**: JWT (HS256), 8-hour expiration
- **Token storage**: localStorage in browser (XSS risk — ⚠️)
- **Password hashing**: bcrypt
- **Authorization model**: Role-based (admin | user)
  - Admin: full access to all resources
  - User (professor): scoped to own questions and lists
- **No OIDC / SSO**

**Default credentials (dev)**: admin / admin123 — must be overridden via secrets.
